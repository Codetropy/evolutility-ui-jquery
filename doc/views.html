Views

All views for a particular Backbone model are defined by a single ui-model.

Evolutility provides 3 types of views:

  • Evol.ViewOne - Views for one model
  • Evol.ViewMany - Views for a collection of many models
  • Evol.ViewAction - Views for actions on a model or a collection

All views are defined at by a single UI model.

Evol.ViewOne

Backbone views for one model.

View

Shows all fields for viewing (read only). Fields are grouped in panels and tabs like the edit view.

new Evol.ViewOne.Edit(myConfig);

 

Edit

Shows all fields for edition. Fields are grouped in panels and tabs.

new Evol.ViewOne.Edit(myConfig);

 

Mini (quick-edit)

Only shows important fields (required or showing as a column in grids) in one panel.

new Evol.ViewOne.Mini(myConfig);

 

JSON

JSON representation of the data.

new Evol.ViewOne.JSON(myConfig);

 

Evol.ViewMany

Backbone views for a collection containing many models.

list

Tabular view of a collection.

new Evol.ViewMany.List(myConfig);

 

cards

Cards view

new Evol.ViewMany.Cards(myConfig);

 

charts

Charts

new Evol.ViewMany.Charts(myConfig);

 

Evol.ViewAction

Filter

Filter collection.

new Evol.ViewAction.Filter(myConfig);

 

Filter example: "phone number start by 415" or "for categories Finances and Business"... 

Export

Export supporting different formats.

new Evol.ViewAction.Export(myConfig);
